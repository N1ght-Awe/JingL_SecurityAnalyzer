---
name: XSS漏洞分析
description: CWE-79 跨站脚本漏洞分析；用于审查前端或服务端中不可信数据进入 HTML、属性、URL、JavaScript 或 CSS 渲染上下文的情况。
---

## 1. 漏洞概述

仅在攻击者可控制数据到达可执行浏览器上下文，且缺少该上下文适配的编码、净化或可信类型约束时确认。区分存储型、反射型和 DOM 型 XSS。

## 2. 危险点清单

- `v-html`、`innerHTML`、`outerHTML`、`document.write`、`dangerouslySetInnerHTML`。
- Thymeleaf `th:utext`、服务端拼接 HTML、模板关闭默认转义。
- 用户可控值进入事件属性、`javascript:` URL、内联脚本或样式。

## 3. 分析步骤

1. 从请求、数据库二次读取、MQ 或富文本字段追踪到具体渲染点。
2. 标明渲染上下文；HTML 文本、属性、URL、JS 和 CSS 不可互相替代防护。
3. 读取净化/编码实现与调用位置，确认它覆盖该字段和该上下文。
4. 追踪存储型链路的写入、读取和展示端；至少给出两层调用链。

## 4. 证据闭合验证

`Source` 为不可信内容；`Propagation` 为字段传递或持久化读取；`Sink` 为危险渲染 API；`Sanitizer` 为上下文适配净化/编码；`Guard` 为访问控制；`Transport` 仅记录跨服务传输；`Preconditions` 为访问页面权限；`Impact` 为受害者会话或业务操作影响。

## 5. 防护措施清单

**强防护：** 默认模板转义；在危险 API 前对同一上下文执行成熟 HTML 净化器；可信类型策略且禁止绕过。

**弱/无效：** 仅替换 `<script>`、黑名单关键字、`replace("<", "")`、只做 URL 编码、仅依赖前端校验。

## 6. 常见误报模式

- 内容为固定字面量或仅来自受控构建产物。
- 危险渲染前对同一字段进行了可验证的上下文净化。
- 字段展示使用正常插值/文本节点而非 HTML 解析。

## 7. 判定标准

确认：可控内容到达危险渲染点，且无覆盖该上下文的强防护。误报：证明内容不可控，或强防护在该路径生效。无法追到写入者/渲染者时标记“缺失关键源码”，不得臆测。

## 8. 置信度评估

高：完整 Source→Sink 链闭合；中：输入或净化实现缺一环；低：仅发现渲染 API。

## 9. 核心纪律

不把“存在 `innerHTML`”直接当漏洞；必须证明数据可控、上下文和防护覆盖范围。
