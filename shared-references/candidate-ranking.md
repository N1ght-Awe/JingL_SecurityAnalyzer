# 候选排序

每条原始命中必须排序。不得静默忽略大量命中集。

## 排序等级

| 等级 | 含义 | 必须处理 |
|------|------|---------|
| P0 | 源和Sink直接或短路径连接，无可见强防护 | 加载对应子Skill并完整审查 |
| P1 | 危险Sink存在，源需追踪或防护不明确 | 加载对应子Skill并继续追踪 |
| P2 | 因枚举/配置/服务间调用/可能的白名单而优先级较低，但未证明安全 | 记录原因，采样或在风险值得时继续 |
| P3 | 仅记录：固定字面量、测试/演示/生成代码、证明不可控、证明有强防护 | 汇总排除原因 |

## SQL `${}` 特殊情况

大量MyBatis `${}` 命中集必须在审查前排序。

P0：
- `${}`接收HTTP/MQ/RPC/导入文件数据
- `${}`在SQL结构位置：WHERE、ORDER BY、GROUP BY、LIMIT、表名、列名、条件片段
- 无枚举、Map映射或强白名单可见
- 源到Sink链直接或短路径

P1：
- `${}` Sink存在但源不明确
- 可能存在外部输入但链不完整
- 无可见强防护

P2：
- 参数来自枚举、配置、固定常量或服务间调用
- 可能存在白名单辅助但实现未读取
- 服务间调用者或参数来源未知

P3：
- SQL片段为固定字面量
- XML include片段为固定内容
- 测试/演示/未部署代码
- 源到Sink证明断裂
- 强白名单映射已证明

服务间接口是前置条件或降级因素，不是自动的误报证明。
