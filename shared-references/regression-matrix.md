# 回归验证矩阵

每次修改扫描索引、子Skill、防护分类、证明模式或主流程后，执行以下最小回归。测试目标是判断和状态，不是攻击执行。

| 场景 | 预期 |
|---|---|
| 18类漏洞各一条Source→Sink直接链 | 进入P0/P1并加载正确子Skill |
| 强防护映射到确切路径 | 误报，附实现证据 |
| 防护名称存在但实现缺失 | 缺失关键源码，不得误报 |
| Source或调用方仓缺失 | 缺失关键源码，附接力指引 |
| `verify=False`且无敏感数据 | 不确认传输凭据泄露 |
| `verify=False`且认证头/Token可追踪 | 生成传输候选并记录反向追踪 |
| 上传仓→制品存储→下载仓→执行 | 生成跨仓供应链候选 |
| P0/P1/P2/P3混合集 | 统计完整，P0/P1不得漏加载子Skill |
| Grep结果超过100条 | 按目录、文件类型、精确模式重试 |
| source-only运行 | 不出现“工具验证确认” |

验证时同时检查：所有索引路由存在对应目录；所有18类有防护条目；供应链有证明模式；主流程引用文件存在。
